## タスク完了時のチェックリスト

コードの変更や記事の執筆が完了したら、以下の項目を確認してください。

### 1. コードの品質と一貫性
- [ ] **フォーマットとLintの確認:**
      `yarn biome:check` を実行し、エラーがでないことを確認する。
- [ ] **スタイルの遵守:**
      `style_and_conventions.md` に記載された規約（ダブルクォート、スペースインデント等）に従っているか目視で確認する。

### 2. ビルドとテスト
- [ ] **コントラクトの変更:**
      `yarn contract compact` (コンパイル) と `yarn contract build` (API生成) を実行し、成功することを確認する。
- [ ] **テストの実行:**
      - `yarn contract test` (コントラクト単体)
      - `yarn cli test-api` (CLIのローカルテスト)
      - 必要に応じて `yarn cli test-against-testnet` (Testnet結合テスト)
      これらを実行し、すべてのテストがパスすることを確認する。

### 3. 記事との整合性
- [ ] **内容の反映:**
      コードの変更が、執筆中の技術記事の内容やコマンド実行例と一致しているか確認する。
- [ ] **手順の再現性:**
      記事に記載する手順（`yarn cli deploy`など）が、変更後も問題なく動作することを確認する。

### 4. 最終確認
- [ ] **コミットの準備:**
      変更内容が単一の関心事であり、コミットメッセージが分かりやすいか確認する。
- [ ] **クリーンな状態:**
      デバッグ用の `console.log` など、不要なコードが残っていないか確認する。
